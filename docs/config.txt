config
==========

rope uses mdconf.  it will try to find out the home directory of the
current user and read `$HOME/.rope`.  (you can pass `-config` to
override this.)

the config should be as follows:

```
# socks

## clientMode
// whether to enable "client mode".
// in client mode, rope act as a "middle man" between you and
// another rope proxy.  the graph would be:
//     you-----rope(client)-----rope(remote)
// this is used w/ clients that does not support SOCKSv5 auth,
// e.g. SwitchyOmega. 
+ enable: [true or false]
+ serverAddr: [server addr]
+ serverPort: [server port]
// only supports RFC1929.
+ username: [username]
+ password: [password]

## v4
// whether to enable socksv4.
+ enable: [true or false]
// bind address.  (127.0.0.1 by default)
+ bindAddr: [bind address]
// bind port.  (1080 by default)
+ bindPort: [bind port]

## v5
// whether to enable socksv4.
+ enable: [true or false]
// bind address.  (127.0.0.1 by default)
+ bindAddr: [bind address]
// bind port.  (1080 by default)
+ bindPort: [bind port]
+ useAuth: [true or false]
+ userDatabase: [path to user database, resolved against home dir]
```
